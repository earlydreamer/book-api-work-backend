## 2. 이번 라운드에서 실제로 잠긴 것

### 2.1 상태 계약과 세션 복원

- mock 세션은 메모리 전용이 아니라 브라우저 저장소에서 복원된다.
- 로그인, 연결 상태, 현재 기록 창, 보관 기록이 새로고침 이후에도 유지된다.
- 페이지는 raw fixture를 직접 읽기보다 컨텍스트와 selector를 통해 파생 상태를 읽는다.

### 2.2 책 자격 기준 단일화

- 책 자격 기준은 `최근 30일 중 기록 20일 이상`으로 통일했다.
- Feed, MakeBook, selector가 같은 기준을 공유한다.
- `15개`, `60개` 같은 임시 숫자 기준은 더 이상 화면 계약으로 쓰지 않는다.

### 2.3 피드 우선순위 재정렬

- 피드는 항상 `오늘 카드 상태`를 1순위로 다룬다.
- 책 관련 정보는 보조 정보로 내려갔다.
- 연결 전, 초대 대기, 오늘 비어 있음, 나만 기록함, 둘 다 기록함 상태에서 상단 CTA가 다르게 바뀐다.

### 2.4 기록 입력 기준

- 기록은 사진이 없어도 남길 수 있다.
- 감정은 필수다.
- 저장하면 현재 연결 cycle의 오늘 카드에 즉시 반영된다.

### 2.5 UX 라이팅 톤

- 앱 안의 문구는 차분하고 부드러운 `~해요.` 체로 통일했다.
- 기준은 토스 UX 라이팅 가이드를 준용한다.
- CTA, 도움말, 빈 상태, 제한 안내까지 같은 톤을 유지한다.

### 2.6 부분 카드 규칙

- 부분 카드는 실패가 아니라 유효한 기록이다.
- 지난 날짜의 부분 카드는 완성 카드와 같은 2열 레이아웃을 유지한다.
- 당일 부분 카드에만 기다림/작성 유도 문구가 나온다.
- 지난 날짜에는 `혼자만 기록을 남겼어요` 상태만 보여주고, 당일 CTA는 보이지 않는다.

## 3. 관계 기록 정책

이번 라운드에서 가장 중요하게 잠근 정책은 아래 4개다.

### 3.1 연결 해제 정책

- 연결을 끊어도 이전 파트너와의 기록은 삭제하지 않는다.
- 현재 연결의 recorded moment는 `archivedMoments`로 이동한다.
- 해제 직후 현재 기록 창은 빈 상태로 초기화된다.

### 3.2 재연결 정책

- 다시 연결하면 기록 일수는 `0일부터 다시 시작`한다.
- 새 연결은 새로운 cycle로 시작한다.
- `startDate`는 재연결 시점의 날짜로 다시 잡는다.
- 이전 연결의 기록은 계속 보관함에 남아 있다.

### 3.3 피드 노출 정책

- 피드는 언제나 `현재 연결`의 기록만 보여준다.
- 새 관계가 시작된 뒤에는 이전 관계 기록이 피드에 노출되면 안 된다.
- 이전 관계 기록은 보관함에서만 확인할 수 있다.

### 3.4 보관함 노출 정책

- 보관함은 `현재 연결 기록`과 `이전 연결 기록`을 시각적으로 분리한다.
- 이전 연결 카드에는 `이전 파트너와의 기록` 라벨을 붙인다.
- 현재 연결과 이전 연결은 섹션 배경, 설명, 개수 배지로도 구분한다.

## 4. 지금 코드에 반영된 구조

### 4.1 세션 구조

```text
MockSession
  - user
  - coupleInfo
  - activeCycleId
  - moments
  - archivedMoments
  - bookStateOverride
```

핵심은 `현재 연결`과 `이전 연결`을 같은 배열로 다루지 않는다는 점이다.

### 4.2 화면별 데이터 원칙

- Feed: `moments`만 사용
- MakeBook: `moments` 기준으로 진행률 계산
- Archive: `moments + archivedMoments`를 분리해서 보여줌
- Me: 연결 해제는 컨텍스트 전이 함수를 통해 처리

### 4.3 선택자 원칙

- 책 자격
- 이번 달 기록 수
- 최근 기록
- 오늘 카드 상태

이 계산은 페이지마다 직접 하지 않고 selector를 거친다.

## 5. 테스트로 잠근 것

현재 테스트로 확인되는 정책은 아래와 같다.

1. 책 자격 계산이 `30일 / 20일` 기준으로 동작한다.
2. 연결 해제 시 이전 기록은 보존되고 현재 기록 수는 초기화된다.
3. 재연결 시 이전 기록은 유지되고 현재 연결은 0일부터 시작한다.
4. 부분 카드의 당일/지난 날짜 UX가 다르게 나온다.
5. 보관함이 `현재 연결 기록 / 이전 연결 기록`으로 나뉜다.
6. 이전 연결 카드에 `이전 파트너와의 기록` 라벨이 붙는다.
7. 새 관계가 생긴 뒤에는 피드에 이전 관계 기록이 노출되지 않는다.

## 7. 이번 라운드 결론

이번 정렬은 디자인 polish보다 `상태 계약 잠금`에 가까웠다.

지금 기준으로 `today-us-front`는 아래 두 가지를 분명히 말할 수 있다.

1. 새 관계가 시작되면 제품의 현재 시간축은 새로 시작한다.
2. 그렇다고 이전 관계의 기록 자산이 사라지지는 않는다.

즉, 이 앱은 `현재 관계의 루프`와 `이전 기록의 보관`을 분리해서 다루는 방향으로 가고 있다.
